OSG场景的组织结构

1. 节点的定义与种类

1.1 BVH树

  • OSG采用包围体包围物体的BVH树来组织场景的节点，每个节点都包含一个包围体，并存储了该节点所包含的物体的索引。
  • BVH树的优点：
    • 快速的空间分割：BVH树通过对物体的空间分布进行分割，将物体划分为多个子节点，使得物体的空间查询和交互变得更加快速。
  • OSG采用Node类来表达一个基础节点
    osg::Node类
    方法 描述
    Node() 构造函数
    void dirtyBound() 提示更新节点的包围体
    const BoundingSphere& getBound() const 获取节点的包围体
    BoundingSphere computeBound() const 虚函数，计算节点的包围体

1.2 节点的父子关系

  • 每个节点都可以有零个或多个子节点，每个节点都可以有一个父节点。
  • 叶子节点仅用来保存准备渲染的实体，其他节点则用来组织场景的结构。
  • ParentList,OSG的场景结构是一种有向无环图,多个父节点可共享同一个子节点.
    Node类的一些接口函数
    方法 描述
    const ParentList& getParents() const 获取节点的父节点列表
    const Group* getParent(unsigned int i) const 获取节点的第i个父节点
    unsigned int getNumParents() const 获取节点的父节点数量
    void addParent(Group* parent) 为当前节点追加一个父节点
    void removeParent(Group* parent) 删除当前节点的某个父节点

1.3 叶节点与组节点

  • 叶节点Geode(Geometry Node):保存和管理一个或者多个可绘制物体的信息,有一个或多个Drawable对象,每个Drawable对象都代表一个可绘制物体.
    osg::Geode类
    方法 描述
    Geode() 构造函数
    bool addDrawable(Drawable*) 从叶节点中追加一个Drawable对象
    bool removeDrawable(Drawable*) 从叶节点中删除一个Drawable对象
    bool replaceDrawable(Drawable* origDraw, Drawable* newDraw) 替换一个Drawable对象
    unsigned int getDrawableIndex(const Drawable*) const 获取Drawable对象在Geode中的索引
    const DrawableList& getDrawableList() const 获取Geode中Drawable对象的列表
  • 组节点Group(Group Node):保存和管理一个或者多个子节点的信息,每个子节点都是一个Node对象.
    osg::Group类
    方法 描述
    Group() 构造函数
    bool addChild(Node*) 从组节点中追加一个子节点
    bool removeChild(Node*) 从组节点中删除一个子节点

2. 节点的访问

2.1 访问器机制

  • 访问器可以向不同的节点元素施加用户自定义的操作,每一个数据元素节点都可以通过accept()方法调用访问器,而访问器通过apply()方法获取传入的节点对象.
    osg::NodeVisitor类
    方法 描述
    NodeVisitor(TraversalMode tm) 构造函数传入参数为节点数的遍历方式:TRAVERSE_ALL_CHILDREN,TRAVERSE_NONE,TRAVERSE_PARENTS
    void traverse(Node& node) 向下一个需要访问的节点推进
    void apply(Node/Group/Geode& node) 对传入的节点对象进行访问,并执行访问器中自定义的节点操作
    void accept(NodeVisitor& nv) 接受一个访问器

2.2 节点的遍历函数

  • OSG中节点访问器对于场景树的访问采用的是深度优先遍历
    osg::Node类
    方法 描述
    void ascend(NodeVisitor& nv) 虚函数,向上一级节点推进访问器
    void traverse(NodeVisitor& nv) 虚函数,向下一级节点推进访问器

3.2 开关节点

  • Switch:在场景运行的某一时刻,可以选择性的显示或隐藏某个节点.
    osg::Switch类
    方法 描述
    Switch() 构造函数
    bool addChild(Node*, bool active=true) 向Switch节点中追加一个子节点,并设置其初始状态
    void setValue(unsigned int pos, bool value) 设置指定索引位置子节点的开关值
    bool getValue(unsigned int pos) const 获取指定索引位置子节点的开关值

3.3 细节层次节点(LOD)

  • LOD:不影响渲染外观的前提下，根据场景对象与观察者的距离，从多个预置方案中选择一种更为简单的方式来表达要渲染的物体，减
    轻系统绘制场景的负担。模型越靠近观察者时越精细，而处于远处时则只需要使用较少的多边形数来表达。
  • LOD的每个子节点都是一种细节层次的表达方案
    osg::LOD类
    方法 描述
    LOD() 构造函数
    void addChild(Node* child, float min, float max) 向LOD节点中追加一个子节点,并设置其对应的观察范围
    void setRange(unsigned int childNo,float min, float max) 设置指定位置的子节点对应的观察范围，或者获取某个子节点对应的观察最小值/最大值
    const RangeList& getRangeListO const 获取所有子节点观察范围的列表
